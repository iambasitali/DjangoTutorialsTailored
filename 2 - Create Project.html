Create Django Project

TO-DO Project

During this tutorial, we’ll guide you on the creation of a basic ToDo application.

We will cover this in two phases:

  • A public site that lets people view TO-DO categories and add items to those categories.
  • An admin site that lets you ADD, UPDATE, and DELETE todo_app.

First thing to make sure that you have Django installed on your system. By running the following command in a shell prompt, you can confirm either Django installed or not along with version specified.

(venv) $ python -m django --version
Note
In all of our lesson we will be giving examples of windows. So, on Mac and Linux instead of pip you have to use pip3 and instead of python you have to use python3.

If Django is installed, an output of the installed version will be displayed. If it isn’t, an error message will occur on screen stating “No module named django”.

If you are using Django for the first time, you’ll have to pay attention on some initial setup. Specifically, you’ll need to auto-generate some code that initiate a Django project – a collection of settings for an instance of Django, including database configuration, initial migrations, Django-specific options and apps related settings. 

(venv) $ django-admin startproject todo

The above-mentioned command will create a folder named "todo" in your current directory.

Let’s look at the tree structure of todo directory:

todo/
  manage.py
  todo/
    __init__.py
    settings.py
    urls.py
    asgi.py
    wsgi.py

These files are:

  • The outer todo/ is the parent directory that holds project like container. You can rename it to anything you like as its name doesn’t matter to Django.
  • manage.py A command-line utility that is most frequently used to interact with this Django project in various ways. You can have a deep look related to manage.py in django-admin and manage.py.
  • The inner todo/ directory is the actual Python package for your project. Its name is the Python package name you’ll need to use to import anything inside it (e.g. todo.urls).
  • todo/__init__.py: An blank file that explain Python that this directory should be considered a Python package. If you stands new to Python, read more about packages in the official Python docs.
  • todo/settings.py: This file contains completed configuration for this Django project. Django settings will display a complete guidance how the variable set in settings.py work at the time of execution.
  • todo/urls.py: The URL declarations for this Django project; a “Complete Route List” of your Django-powered site.
  • todo/asgi.py: An entry-point for ASGI-compatible web servers to serve your project.
  • todo/wsgi.py: An entry-point for WSGI-compatible web servers to serve your project.

The development server

Let’s verify your Django project works. To start the server move to parent directory(todo) and run the following commands:

(venv) $ python manage.py runservers

You’ll see the following output on the command line:

Performing system checks...

System check identified no issues (0 silenced).

You have unapplied migrations; your app may not work properly until they are applied.
Run 'python manage.py migrate' to apply them.

August 13, 2021 - 15:50:53
Django version 3.2, using settings 'todo.settings'
Starting development server at http://127.0.0.1:8000/
Quit the server with CONTROL-C.

Look at the above screen, It means Django has started its services on localhost and default port is 8000.

Now the server is running, you can copy this link to the browser http://127.0.0.1:8000/. You’ll see a “Congratulations!” page, with a rocket taking off. It worked!
