Create First Django App

TO-DO App

Every Django project can have more than one Django apps. By executing the startproject command in the previous section, an project container is build that is compulsory for every Django project. After creating project, next-step is to create one or more Django app that’ll contain the specific functionality of your web application.

You can execute the startapp command through the manage.py file for creating an app:

$ python manage.py startapp <appname>

The startapp command create a default folder structure for a Django app. This tutorial uses todo_app as the name for the app:

$ python manage.py startapp todo_app

Once the app is created using startapp command, you’ll see that another folder is added to your Django folder structure.

todo/
  -todo_app/
     migrations/
         __init__.py
     __init__.py
     admin.py
     apps.py
     models.py
     tests.py
     views.py
  -todo/
     __init__.py
     settings.py
     urls.py
     asgi.py
     wsgi.py
  -manage.py

In this tutorial, the newly created folder is named as "todo_app" used when running the command. You can see there are a number of .py files resides in the app direcotry.

This Django app folder is the main folder which act as working directory when creating your web application. You’ll also need to make some changes in the management app, todo/, but you’ll build most of your functionality inside the Django app, todo_app.

Move to your todo_app/ folder and open up the newly generated Python files. Go through the code comments in each file and see if they can help you to create a clear picture of each use case.

Here are the 3 most notable files that were created in the app directory:

  • __init__.py: Python uses this file to declare a folder as a package, which allows Django to use code from different apps to use the overall features of your web application.
  • models.py: As Django follows ORM approach so models.py is used to declare app's models, which allow web application to interact with the database.
  • views.py: Most of the code logic need to be write in this File as it act like controller of whole application.
Important!
If Django is not aware about todo_app, these steps we do later won’t work. So add 'todo_app' to INSTALLED_APPS. The settings module will be in todo/settings.py.

INSTALLED_APPS = [
'myapi.apps.MyapiConfig',

... # Leave all the other INSTALLED_APPS

]

At this point, you have done setting up an Django application and you can start implementing your ideas. From here on out, it’s up to you what you want to build to create your own unique project.
