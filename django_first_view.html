Django View

Django View:-

In the Django framework, views are Python functions or classes that receive a web request and return a web response. The response can be a simple HTTP response, an HTML template response, or an HTTP redirect response that redirects a user to another page.

Let's write our first view

Open todo_app/views.py and write the following code in it:

todo_app/views.py 
                                from django.http import HttpResponse
                                
                                def index(request):
                                    return HttpResponse("Hello, Django. You're at the Todo index.")

Now that we have written our view we need to call that. To do so we will map this view with a URL route.

To create a URLconf you have to create a file urls.py inside todo_app directory.

After that put following code init:

todo_app/urls.py 
                                from django.urls import path
                                
                                from . import views
                                
                                urlpatterns = [
                                    path('', views.index, name='index'),
                                ]

This URLconf is App level URLconf. We need to add these URLconf into our project level URLconf. Todo so, we need to include a path to these configurations in our project urls.py. Open todo/urls.py and add following code in it:

todo/urls.py 
                                from django.contrib import admin
                                from django.urls import include, path
                                
                                urlpatterns = [
                                    path('todo_app/', include('todo_app.urls')),
                                    path('admin/', admin.site.urls),
                                ]

The include() function allow us to refer to other URLconfs.

The idea behind include() is to make it easy to plug-and-play URLs. Since todo_app are in their own URLconf (todo_app/urls.py), they can be placed under “/todo_app/”, or any other path root, and the app will still work.

We are complete here adding a view now to verify the working run server with following command: 

                                python manage.py runserver

Once the server starts open your browser and goto localhost:8000/todo_app/ , and you should see the text “Hello, Django. You’re at the Todo index.”, which you defined in the index view.

The path() function in our URLconf is passed four argument two of which are required and two are optional. All four arguments are explained below:

path() argument: route

route is a string that contains URL pattern. When a request comes django start matching the pattern of the request with the patterns defined in our URLconf both project level and app level. For example in our case localhost:8000/todo_app/ the django server will only look for todo_app/. It will omit the server name.

path() argument: view

When django finds a matching pattern it will call the specified view function with an HTTPRequest parameter and also all the keyword arguments which are captured with the request. These arguments are explained below.

path() argument: kwargs

In simple words this is the extra information passed to the view. It is in the form of python dictionary with key:value pairs.

path() argument: name

The name argument of the url lets you access the url unambiguously from anywhere in django. We will see example of this in details in templates.
